⚠️ 1. Tanpa Generic

Class Box menggunakan dynamic → bisa diisi tipe apa saja, TIDAK AMAN (runtime error berpotensi).

class Box {
  dynamic data;
  Box(this.data);
}
void main() {
  var box = Box(42);
  print(box.data);
  box.data = "Hello"; // Bisa diubah jadi String
}

✅ 2. Dengan Generic

Box<T> → tipe ditentukan saat instansiasi, AMAN (compile-time error jika salah tipe).

class Box<T> {
  T data;
  Box(this.data);
}
void main() {
  var intBox = Box<int>(42);
  var stringBox = Box<String>("Hello");
  print(intBox.data);
}

💻 3. Sintaks Generic Class & Function

class MyClass<T> {
  T data;
  MyClass(this.data);
  T getData() => data;
}
T myFunc<T>(T value) => value;
class Pair<K, V> {
  K key; V value;
  Pair(this.key, this.value);
}

📦 4. Kelas Box<T> dengan Getter/Setter

class Box<T> {
  T? _value;
  Box(this._value);
  T? get value => _value;
  void setValue(T value) => _value = value;
}

🔑 5. Multiple Type Parameters (Pair)

class Pair<K, V> {
  K key; V value;
  Pair(this.key, this.value);
  void printPair() => print("Key: $key, Value: $value");
}
void main() {
  var pair = Pair<String, int>("umur", 20);
  pair.printPair();
}

🔗 6. Fungsi Generic Combine 2 Tipe

String combine<T, U>(T value1, U value2) => '$value1, $value2';
void main() {
  print(combine<String, int>("Umur", 20));
  print(combine<int, double>(100, 99.5));
}

🛡️ 7. Generic dengan Extends (Batas Tipe)

<T extends num> → hanya menerima int, double, num.

class NumberBox<T extends num> {
  T value;
  NumberBox(this.value);
  T getValue() => value;
}
// NumberBox('test'); // ERROR!

🎯 8. Type Inference (Inferensi Tipe)

T myFunc<T>(T value) => value;
void main() {
  var a = myFunc(10);     // Dart inferensi sebagai int
  var b = myFunc("test"); // Dart inferensi sebagai String
  print(a.runtimeType);   // int
  print(b.runtimeType);   // String
}

🔒 9. Pembatasan Tipe (Interface Iterable)

void printIterable<T extends Iterable<dynamic>>(T items) {
  for (var item in items) print(item);
}
void main() {
  printIterable<List<int>>([1, 2, 3]);
  // printIterable(10); // ERROR!
}

🐪 10. Pembatasan dengan Abstract Class

abstract class Animal { void makeSound(); }
class Dog implements Animal { void makeSound() => print("Woof!"); }
class AnimalCage<T extends Animal> {
  T animal;
  AnimalCage(this.animal);
  void sound() => animal.makeSound();
}

💼 11. Praktik: Repository, Swap, Triple

class Repository<T> {
  final List<T> _items = [];
  void save(T item) => _items.add(item);
  List<T> getAll() => _items;
}
void swap<T>(List<T> list, int i, int j) {
  T temp = list[i]; list[i] = list[j]; list[j] = temp;
}
class Triple<A, B, C> { A first; B second; C third; }

🗄️ 13. Studi Kasus: Cache Generic

class Cache<T> {
  final Map<String, T> _storage = {};
  void set(String key, T value) => _storage[key] = value;
  T? get(String key) => _storage[key];
}
class Product { String name; int price; }
void main() {
  var cache = Cache<Product>();
  cache.set("p1", Product("Laptop", 1000));
}

📝 14. Latihan: Stack, MergeLists, Response

class Stack<T> {
  final List<T> _items = [];
  void push(T item) => _items.add(item);
  T pop() => _items.removeLast();
  T peek() => _items.last;
}
List<T> mergeLists<T>(List<T> a, List<T> b) => [...a, ...b];
class Response<T> {
  T? data; String? error;
  Response.success(this.data) : error = null;
  Response.error(this.error) : data = null;
}

✨ Ringkasan Manfaat Generic

  • Type Safety - Error terdeteksi saat compile-time
  • Code Reusability - Satu class untuk berbagai tipe data
  • No Casting - Tidak perlu konversi tipe manual
  • Performance - Lebih optimal dari dynamic
  • Constraints - Bisa batasi tipe dengan extends

📌 Kesimpulan

  • ✅ Generic membuat kode TYPE-SAFE dan REUSABLE
  • ✅ Bisa digunakan di class, fungsi, dan collections
  • ✅ Type inference memudahkan penggunaan
  • ✅ Constraints (extends) membatasi tipe yang diperbolehkan
